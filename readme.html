Welcome to Offline Registration Portal 2.0 – your one-stop solution for event registrations without the hassle of online connectivity! Whether you're an administrator configuring the event or a user registering for an event, this guide will help you navigate through the system like a pro. Let’s get started! 🚀

📌 Features of Offline Registration Portal 2.0

  • ✅ Fully Offline Functionality – No internet? No problem! All data is stored in your browser’s local storage.
  • ✅ Admin Panel for Customization – Set up event names, categories, and team sizes effortlessly.
  • ✅ Dynamic Registration System – Supports individual and team-based registrations.
  • ✅ Auto-Saving – Your data is never lost, even if you refresh or close the browser.
  • ✅ Export & Print Options – Download the registration data as an Excel file or print them directly.
  • ✅ Live Counter – See the number of registrations for each event in real time.
  • ✅ Starting Point Configuration – Set a unique registration ID starting number.
  • ✅ Reset Feature – Need a fresh start? Reset everything with one click!
  • ✅ Timestamp for Registrations – Each registration includes a timestamp in mm dd yyyy along with time.

🛠️ Admin Panel: How to Use

(This is where the magic begins! ✨)

The Admin Panel allows you to configure the event settings before the main registration starts. Here’s how to use it:

1️⃣ General Settings

  • Enter the title and subtitle for your event.
  • These will appear on the main registration page.

2️⃣ Event Configuration

You can configure three event types:

  • 🛠 Tech Events – Coding competitions, bug bounty, etc.
  • 📊 Management Events – Business simulations, case studies, etc.
  • 🌍 Common Events – Open to all participants (e.g., quizzes, general competitions).

Adding Events

  • Each category has at least one default event (which cannot be deleted).
  • You can add more events and specify the team size for each.
  • The event names are dynamically numbered (e.g., 1st Item Name, 2nd Item Name, etc.).
  • To delete an event, click "Delete" (but the first event is non-deletable).

3️⃣ Setting the Starting Number

  • Enter a three-digit starting number (e.g., 101, 201, etc.).
  • This will be used for auto-generating unique IDs for registrations.
  • The placeholder will always show the current saved starting number.

4️⃣ Finalizing the Setup

  • Click "Final Submit and Load Registration Portal", and you’re done! 🎉

5️⃣ Reset Everything (The Nuclear Button ☢️)

  • A big red button at the bottom allows you to reset everything (configurations & registrations).
  • Use this only if you want a fresh start – it clears local storage and refreshes the page.

📝 Main Registration Portal: How to Use

(This is where people sign up for events! 📝)

1️⃣ Choose an Event Type

  • Select Tech, Management, or Common.

2️⃣ Choose an Event Item

  • Based on the event type, select an event.

3️⃣ Enter Participant Details

  • Team members' names (auto-filled based on team size).
  • College name.
  • Phone number.
  • Email ID.
  • Payment details.

4️⃣ Mode of Registration

  • Select either "Spot" (default) or "Online".

5️⃣ Submit the Form

  • Your registration will be saved!

Extra Cool Features 🎉

  • Auto-fill options for college name & phone number.
  • Live Registration Counter – View the count of registered participants per event.
  • Scroll Buttons – Quickly navigate up & down the page.
  • Timestamp for Registrations – Each registration includes a timestamp in mm dd yyyy format, along with the time of registration.

📤 Export & Print Options

🔹 Export to Excel

  • Download all registration data as an Excel file.
  • The Excel file includes all details, such as event name, participant names, phone numbers, college, email, payment, and timestamp.

🔹 Print by Event Type

  • Print data filtered by event type.
  • The printed output includes a formatted table with event details, participant names, phone numbers, and timestamps.

🔹 Smart Formatting

  • Long participant names wrap inside the table automatically (no cut-off text!).

🔄 Data Management

  • Data is auto-saved in your browser’s local storage.
  • Deleting an entry updates the starting point automatically, allowing the next registration to reuse the deleted entry's ID.
  • Reset button clears everything (use wisely!).

🎯 Final Thoughts

This portal is designed to make event registration fast, easy, and reliable – even without the internet! 💡 Whether you're managing the event or participating in it, Offline Registration Portal 2.0 has got you covered.

🚀 Now go ahead, set up your event, and start registering participants hassle-free! 🚀

Additional Notes for Administrators

  • Timestamp Format: The timestamp is displayed in mm dd yyyy format (e.g., 10 25 2024), don't get confused.
  • ID Reuse: When an entry is deleted, the system automatically reuses the deleted entry's ID for the next registration.
  • Backup Data: Always export the data to Excel before resetting the system or clearing local storage.
